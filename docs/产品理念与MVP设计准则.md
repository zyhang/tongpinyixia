# 答案之外｜产品理念与 MVP 设计准则

> 状态：已确认。后续产品、设计、开发与文案工作均须先遵守本文件；如与历史设计稿或旧命名冲突，以本文件和《答案之外-品牌文案规范》为准。

## 1. 产品是什么

**答案之外**不是交友、聊天、测试人格或实时匹配产品。

它提供一次轻量、匿名、没有负担的理解体验：用户选一个更像自己的答案，看见真实比例，再读到做出相同选择的人留下的真实理由。

品牌主张：**答案只有两个，理由远不止两种。**

产品的价值不在于把人“配对”，而在于让用户发现：相同选择背后可能有不同经历；有些想法不需要认识彼此，也可以被理解。

## 2. 核心体验与主循环

MVP 的主循环应始终轻量、可中断、不产生任务感：

```text
主循环：今日一问 → 选择一个答案 → 看真实比例与一句真实理由 → 再看一道题
可选支线：选择结果 → 我也想说说为什么 → 让这句话飘进答案之外 → 再看一道题
```

- 题目只有两个选项；没有标准答案，只选此刻更像自己的那个。
- 用户作答后，**选择比例与匿名理由在同一页连续呈现**，不得再用“看看他们怎么想”作为进入理由内容的额外门槛。
- “再看一道题”是结果页最明确的主操作；不设置固定题量、连胜、进度条或完成一套题的压力。
- 用户可随时停止，不需要解释、不需要留下内容，也不应被引导完成额外动作。
- 理由投稿是可选支线，不能成为进入下一题或查看理由内容的门槛。

## 3. MVP 范围与明确不做

首期 MVP 的基础循环是“选择—结果—真实理由—下一题”，并开放受控的匿名理由投稿支线。

以下能力不在 MVP 中，界面、按钮、文案和数据模型均不应暗示其存在：

- 分享卡、保存图片、朋友圈/抖音分享等外部传播功能。
- 私信、关注、主页、匹配、好友、实时在线或任何关系建立能力。
- “我的回声”“我的理由”“可收回”“历史记录”等会让用户担心个人内容被长期留存的功能或暗示。

理由投稿是匿名、轻量的可选表达，不提供个人投稿记录、历史列表或撤回入口，也不得由此暗示内容会长期沉淀在用户账户中。用户界面不使用审核、授权、提交等平台化措辞，也不要求额外勾选确认；内容治理如有需要，应在后台处理，不能转化为用户负担。

## 4. 真实、匿名与安全边界

- 比例、作答人数和更新时间只能在真实数据可用时展示；无可靠数据时隐藏或使用规范中的数据不足状态，禁止静态伪数据。
- 理由只能来自已授权、经审核的真人匿名理由；理由不足时宁可少展示，不能用模板或 AI 文案伪装真人理由。
- 不使用“匹配成功”“正在寻找同频的人”“有人正在等你”等实时匹配或关系建立暗示。
- 统一安全边界文案：**不展示身份 · 不建立联系 · 只留下真实想法**。

## 5. 结果页与理由交互

结果页的信息优先级：

1. 用户选择确认。
2. 真实比例及其数据口径。
3. 一句来自匿名理由库的真实理由。
4. 再看一道题。

理由投稿页及成功页的文案以品牌规范为准，但应维持“写一句、轻轻放进去、随缘被遇见”的低压力表达；投稿成功后，主操作仍应是“再看一道题”。

理由卡内允许两种低层级操作：

- **这句话说中了我**：一次性轻反馈。点击后显示“原来你也这样想”，再显示有多少人也被这句话打动；人数不得在点击前提前影响用户判断。该反馈不提供取消。
- **再看一种想法**：仅替换当前理由卡内容，不刷新整页、不改变用户选择、不暗示实时匹配。

理由卡操作不得与“再看一道题”争夺主操作层级，不应做成与主按钮同等醒目的大按钮。

## 6. 文案原则与命名

品牌与页面文案以 [《答案之外-品牌文案规范》](./答案之外-品牌文案规范.md) 为唯一标准。

- 产品名：**答案之外**。
- 品牌副标题：**选择之后，听见真实的想法**。
- 首屏标签：**今日一问**。
- 语言应简洁、真实、有温度，不过度煽情；少用“实验、共鸣、匹配、投稿”等产品术语。
- 不替用户判断人格、情绪或价值观；不把“选择相同”夸大成“完全理解”或“高度匹配”。
- 任何新增文案应先说明用户能得到什么，再补充必要规则；提示越少越好，用户应能一目了然地完成操作。

## 7. 视觉与交互气质

- 面向夜间轻浏览场景，视觉应安静、克制、有一点私密氛围，而不是高刺激或游戏化。
- 深色方向可以使用低饱和中性色作为主体，以有限颜色承担信息与情绪强调；避免过度炫光、宇宙、雷达、人群匹配等会误导产品属性的视觉隐喻。
- 一屏只突出一个主要动作；说明、来源、边界和轻交互都应降级呈现。
- 输入或内容展示必须有足够的文字对比度、字号和触控面积；不得为了氛围牺牲可读性。
- 优先做减法：重复提示、重复确认、无后续价值的互动和未进入 MVP 的入口都应移除。

## 8. 实施前检查

每次新增或修改页面、功能、文案时，先回答：

1. 它是否服务于“选择—理解理由—继续轻松作答”的主循环？
2. 它是否把匿名理解误导为社交、匹配、留存或内容生产？
3. 它是否承诺了当前 MVP 没有实现的数据、审核、记录、撤回或分享能力？
4. 它是否符合品牌文案规范，并且比现有方案更少负担、更自然？

任一答案为否，先停止实现，回到产品目标重新确认。
