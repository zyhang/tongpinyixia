## 1. 品牌定稿

### 1.1 产品名

**答案之外**

### 1.2 品牌副标题

**选择之后，听见真实的想法**

### 1.3 品牌主张

**答案只有两个，理由远不止两种。**

### 1.4 一句话介绍

选一个更像你的答案，看看有多少人和你一样，再听听他们为什么这样选。

### 1.5 三句核心文案及职责

| 文案 | 使用位置 | 主要职责 |
|---|---|---|
| 选择之后，听见真实的想法 | 品牌名下方、产品介绍 | 解释产品 |
| 答案只有两个，理由远不止两种。 | 启动页、海报、运营素材 | 建立品牌记忆 |
| 看看答案之外 | 结果页主按钮 | 推动用户进入理由内容 |

## 2. 文案原则

### 2.1 先表达用户收益

品牌第一视觉层不再同时堆叠“轻量、匿名、共鸣、实验”等抽象概念，而是直接告诉用户：作出选择后，可以看见真实比例并听见真实理由。

### 2.2 保持真实克制

- 只在数据真实可用时展示比例、人数和更新时间。
- 只展示经过授权和审核的真人理由。
- 理由不足时宁可少展示，不用模板或 AI 内容冒充真人理由。
- 不使用“正在寻找同频的人”“匹配成功”等实时匹配暗示。

### 2.3 不暗示关系建立

产品提供的是一次性的理解和共鸣，不是交友、聊天或持续匹配。身份与关系边界优先由匿名标识、无关系入口和整体体验自然传达，避免使用清单式声明反复解释。

页面确有需要时，低调使用品牌金句：

**不需要认识，也可以理解一句话。**

### 2.4 语言风格

- 简洁、真实、有温度，但不过度煽情。
- 少用产品术语和实验术语，多用用户能够自然说出口的话。
- 避免替用户判断人格、情绪或价值观。
- 避免把“选择相同”夸大为“完全理解”或“高度匹配”。

## 3. 首屏文案

### 3.1 推荐版本

| 元素 | 文案 |
|---|---|
| 产品名 | 答案之外 |
| 品牌副标题 | 选择之后，听见真实的想法 |
| 栏目标签 | 今日一问 |
| 题目引导 | 如果是你，会怎么选？ |
| 作答提示 | 没有标准答案，只选此刻更像你的那个。 |
| 作答量 | 首屏不展示 |
| 匿名标识 | 在品牌标题旁低调显示「匿名」 |
| 题目共情句 | 随题目变化；本题使用“想接住一个人的难过，本来就不只一种方式。” |

首屏不展示作答人数，避免在用户选择前制造从众暗示，也不让冷启动规模成为页面重点。

匿名性应通过稳定的产品标识与整体体验持续传达，不在题目卡或页面底部重复解释。题目卡底部优先放与当前问题相关、且不偏向任一选项的共情文案；没有合适文案时宁可留白。

## 4. 选择结果页文案

### 4.1 选择确认

**你选择了「A 陪他一起骂」**

### 4.2 核心结果

**有 27% 的作答者，和你选了一样**

数据说明：

**基于 1,284 次作答**

该说明只在样本达到展示门槛后出现，并作为低层级数据说明；样本不足时不展示百分比，直接使用数据不足状态。“有效作答”等后台术语不进入用户界面。

### 4.3 理由呈现

不额外添加内容引导，直接呈现理由卡。选择结果、理由正文与来源说明已能自然说明内容。

主按钮：

**看看答案之外**

备选按钮：

- 听听他们怎么想
- 看看他们为什么这样选
- 读一句同选者的理由

正式版本优先使用“看看答案之外”，让功能入口同时承担品牌强化作用。

| 内容引导 | 一个和你选择相同的人这样想： |

| 换一条理由 | 再看一种想法 |
| 点击共鸣后的反馈 | 原来你也这样想 |
| 氛围金句 | 不需要认识，也可以理解一句话。 |

理由正文示例：

> “有时候他需要的不是建议，而是有人先和他站在一起。”

## 6. 数据不足与异常状态

### 6.1 样本不足

标题：

**答案还在慢慢聚集**

说明：

**等更多人作答后，再回来看看。**

### 6.2 理由不足

标题：

**这个答案背后，还没有足够多的真实理由**

说明：

**我们宁愿暂时少展示，也不替任何人编造想法。**

### 6.3 加载失败

标题：

**暂时没能打开答案之外**

按钮：

**再试一次**

异常状态不得使用“正在匹配”“正在寻找同频的人”等容易造成实时匹配误解的表达。

## 7. 理由投稿入口

> 当前 MVP 开放理由投稿入口。它是一条可选的匿名表达支线：不提供个人投稿记录或分享能力，也不在用户界面强调审核、授权和内容处理流程。

入口标题：

**你的答案背后，也有一个为什么吗？**

辅助说明：

**留下一句话。也许哪天，会被作出相同选择的人碰见。**

入口按钮：

**我也想说说为什么**

空间有限时可使用：

**我也想留一句**

| 写作提示 | 一句真实的话就够了。 |
| 提交按钮 | 让这句话飘进答案之外 |

## 9. 投稿成功页文案

成功标题：

**好啦，这句话飘出去了**

流程说明：

**也许很快，也许过一阵，它会被某个同样选择的人碰见。**

情绪收束：

**随缘，不催。**

操作按钮：

- 主按钮：**再看一道题**
- 次按钮：**今天先到这，收工**

## 10. 非 MVP 预留｜分享卡文案

> 当前 MVP 不提供分享卡、保存图片或外部分享能力。本节仅保留为未来能力评估时的文案储备，不得出现在当前页面、线框图或验收范围中。

| 元素 | 文案 |
|---|---|
| 品牌名 | 答案之外 |
| 卡片主标题 | 我的答案是 A |
| 结果信息 | 本题有 27% 的作答者和我一样 |
| 分享引导 | 如果是你，会怎么选？ |

卡片情绪文案可根据版式选择：

- 答案只有两个，理由远不止两种。
- 这是我的答案，你的呢？
- 我们选得一样，也可能有不同的为什么。
- 选择让我们站在一边，理由让我们看见彼此。

分享卡不得展示他人的原始理由、身份信息或未经单独授权的内容。

## 12. 品牌语句库

以下文案可用于启动页、海报和运营素材：

- 答案只有两个，理由远不止两种。
- 选择让人站在一边，理由让人彼此看见。
- 你选择了什么，也许没有你为什么选择更重要。
- 相同的答案背后，住着不同的人生经验。
- 先选一个答案，再听听答案之外。
- 有些想法说出来以后，才发现并不孤单。
- 不需要认识，也可以理解一句话。
- 我们不替你定义答案，只让真实的理由被看见。
