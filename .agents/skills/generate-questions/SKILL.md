# 题库扩题助手

为「答案之外」生成符合规范的新题目、选项与匿名理由，并写入 `题库/题库.csv` 与 `题库/理由.csv`。

## 参数

- `theme`: 主题/场景，如 `前任`、`吵架`、`催婚`
- `angle`: （可选）角度/矛盾，如 `行为选择`、`第一时间反应`、`事后处理方式`
- `count`: 需要生成的题目数量，默认 3
- `source_hint`: （可选）灵感来源链接或关键词

## 执行步骤

### 1. 读取规范与题库
- 读取 `docs/产品理念与MVP设计准则.md`
- 读取 `docs/答案之外-品牌文案规范.md`
- 读取 `题库/好题标准.md`
- 读取 `题库/题库.csv`，统计当前类别分布，生成「已占用场景清单」

### 2. 社交平台取材
- 根据 `theme` 和 `source_hint`，尝试用 `web_fetch` 抓取公开讨论页面
- 可尝试的公开来源：
  - 百度贴吧：`https://tieba.baidu.com/f?kw=...`
  - 豆瓣小组公开帖
  - 微博话题公开页面
- 如果抓取失败，降级为：基于 `theme` + `angle` + 真实生活经验生成候选题材，并标注「未经过真实社交平台验证」

### 3. 生成候选题材
- 从抓取内容或经验中，提炼 5-10 个候选题材/角度
- 每个题材必须是真实人生中人们会纠结、会站队的具体场景

### 4. 去重筛选
- 对每个候选题材，对照「已占用场景清单」检查：
  - 场景前缀是否重复
  - 核心矛盾是否重复（主动 vs 被动、先A还是先B、独处 vs 社交等）
  - 选项结构是否重复
- 重复或近似的题材丢弃或改写

### 5. 生成题目
对通过的题材，生成：
- `question_text`：符合「一秒题感」，自己有答案
- `option_a` / `option_b`：双方都有对的逻辑，具体有画面感
- `category`：优先情感关系类；如题材属于其他类，也允许
- `empathy_text`：不偏向任一选项，简洁有温度

检查是否踩中「七种必死题」：
- 需要想想
- 一边明显对
- 太抽象
- 需要知识
- 诱导高低
- 假互斥
- 对谁都一样

### 6. 类别分布控制
- 统计当前情感关系类占比
- 如果新增题目后会导致情感关系类低于 40%，优先：
  - 将可改写为情感关系类的题材改写
  - 或减少非情感类题目数量
- 如果无法调整，输出警告并停止写入

### 7. 生成理由
- 每道通过的题为 A、B 两个选项各生成 5 条理由
- 理由必须像**真人匿名投稿**，而不是 AI 总结或第三方评论：
  - **短**：控制在 15–40 字，一句话说完
  - **有个人立场**：不中立、不客观，是我怎么想的
  - **带个人经历或具体情境**：能闻到「我经历过」的味道
  - **第一人称自然口吻**：像随口说出来的一句话
  - **避免**：说教、道理、概括、「应该」「其实」「我觉得关键是」等评论腔

**风格对比**：

| ❌ 不像真人投稿 | ✅ 像真人投稿 |
|---|---|
| 复合需要慎重，过去的问题可能重现。 | 复合过一次，同样的问题又吵了一遍，我再也不想绕回去了。 |
| 看到前任有新恋情，应该尽快放下。 | 看到他新发的那条动态，我盯着看了三遍，然后把自己关进厕所冷静。 |
| 删联系方式有助于走出一段感情。 | 删完那天晚上我把聊天记录恢复了五次，第六次才真正死心。 |

### 8. 预览与写入
- 输出新增题目预览表（question_text / option_a / option_b / category / empathy_text）
- 输出理由样例（每题各 2 条）
- 追加写入：
  - `题库/题库.csv`：id 从当前最大 id +1 开始连续递增
  - `题库/理由.csv`：id 从当前最大 id +1 开始连续递增，question_id 对应新题 id
- 写入后重新统计类别分布并报告

## 输出格式

新增题目预览：

| id | 题面 | A | B | 类别 | 共情文案 |
|---|---|---|---|---|---|

理由样例：

| question_id | 选项 | 理由 |
|---|---|---|

写入确认：

- 已写入 `题库/题库.csv`：N 题
- 已写入 `题库/理由.csv`：M 条
- 当前情感关系类占比：X%
